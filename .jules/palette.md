## 2026-03-10 - Visually Hidden Interactive Elements Accessibility
**Learning:** When using CSS patterns like `opacity-0 group-hover:opacity-100` to hide interactive elements until hovered, it completely breaks keyboard accessibility because the elements remain invisible (or sometimes un-focusable depending on other attributes) when navigating via tab. Moreover, icon-only buttons lacking `aria-label`s provide no context to screen readers, turning hidden action buttons into an accessibility double-fault.
**Action:** Always pair `opacity-0 group-hover:opacity-100` with `focus-within:opacity-100` on the container so that hidden elements become visible during keyboard navigation. Additionally, mandate `aria-label` and `focus-visible:ring-2 focus-visible:outline-none` on all icon-only buttons to ensure both visual and screen-reader accessibility.
