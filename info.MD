Assignment Objective 

The purpose of this assignment is to evaluate your practical experience in building a full-stack application using the MERN stack (MongoDB, Express.js, React.js/Next.js, Node.js). 

You are required to build a small application that allows users to create and manage support tickets. 

The focus is on: 

Code quality 

Application structure 

API design 

Database modeling 

UI implementation 

Overall problem-solving approach 

The solution does not need to be production-ready; clarity and correctness are more important than perfection. 

Shape 

Problem Statement 

Build a simple Support Ticket Management Application where users can: 

Create tickets 

View ticket list 

Update ticket status 

Delete tickets 

Filter tickets based on status or priority 

Authentication or user management is not required. 

Shape 

Functional Requirements 

1. Ticket Creation 

Users should be able to create a ticket with: 

Ticket Title 

Description 

Priority (Low / Medium / High) 

Status (Open by default) 

Shape 

2. Ticket Listing 

Display all tickets with: 

Title 

Description (short preview) 

Priority 

Status 

Created Date 

Shape 

3. Update Ticket 

User should be able to: 

Change ticket status 

Change ticket priority 

Shape 

4. Delete Ticket 

Users should be able to delete a ticket. 

Shape 

5. Filter Tickets 

Provide filters for: 

Ticket status 

Ticket priority 

Optional: Search by ticket title. 

Shape 

Technical Requirements 

Backend (Node.js + Express) 

Create REST APIs for ticket operations. 

Use MongoDB to store ticket data. 

Organize code into controllers, routes, and models. 

Include basic input validation. 

Handle errors properly. 

Shape 

Frontend (React/ Next.js) 

Dashboard on Render Page. 

Create pages/components to: 

Create ticket 

View ticket list 

Edit ticket 

Use API calls to fetch/update data. 

Provide loading and error states. 

Keep UI simple but functional. 

Shape 

Database (MongoDB) 

Minimum fields in Ticket collection: 

title 

description 

priority 

status 

createdAt 

Additional fields may be added if required.

Application source code (frontend + backend).

Any assumptions or limitations documented.
